# 语言风格画像

## 总体判断

该版本呈现明显的解释性和学习型风格。译者倾向于把巴利语中的词源、语法关系、隐含主语或语义关系显性化，而不是单纯追求流畅汉语。

## 主要风格特征

### 保留巴利词形

译文常在中文译名后保留巴利词，例如 `勤学者(vedeha)`、`茧丝(koseyya)`、`变麻(sāṇa)`。这说明该版本重视术语可追溯性，适合用于术语抽取和词源解释分析。

### 使用解释性连接词

高频中文信号包括：

- `即`
- `因此`
- `由于`
- `称为`
- `所谓`
- `以...义`
- `故为`

这些词常用来显化巴利注释书中的定义、词源、因果和命名关系。

### 使用方括号补足信息

方括号用于补出隐含主体、语法说明、地名说明或译者不确定内容。例如 `[很多人]`、`[他]`、`[额外]有[字母]`。这类内容应在结构化时标记为 `translator_supplement`，避免与原文直接等同。

### 换行拆分并列解释

该版本常用换行拆分多重词源解释或并列义项。程序处理时不应简单删除换行；换行可能表示译者对解释层次的划分。

## 风格风险

- 译文中解释性补足较多，不能把所有中文内容都视为原文逐词对应。
- 夹注和括号中的巴利词对术语抽取有价值，但会给普通中文分词造成噪声。
- 学习型译法有时牺牲汉语自然度，不能直接当作通用出版译文风格。
- 方括号中可能包含译者推测，应与正文译文分层处理。

## RAG 建议

检索该版本时，应优先保留：

- Pali 原词；
- 中文译词；
- 括号内巴利词；
- 方括号补足；
- `category` 和 `path` 上下文；
- 原始 `unit_id`。
