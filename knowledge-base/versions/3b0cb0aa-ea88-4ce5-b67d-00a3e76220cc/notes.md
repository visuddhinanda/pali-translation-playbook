# 研究笔记

## 本轮试点范围

本轮只对 `胜嗣Jinadāyāda` 版本做初步画像和少量样例条目。目的是验证知识库结构，而不是完成该版本的全量抽取。

## 后续应重点复核

- `new translate the` 等英文片段是否来自残留机器翻译或特殊格式。
- 繁简混用现象，例如部分样例中出现 `將`、`淨`、`說`。
- 方括号中的内容是否全部应归类为译者增补。
- `paccayo` 是否应统一译为 `后缀`，还是要区分语法术语中的 `缘`。
- 绝对分词结构的样例需要人工筛选，不能只靠后缀正则判断。

## 大文件处理建议

后续分析大文件时不要一次性生成完整知识库。建议流程：

1. 先生成版本统计画像。
2. 按 `path` 或 `category` 分块。
3. 每次只抽取一个类别，如术语或句式。
4. 每个条目必须有证据 `unit_key`。
5. 人工确认格式后再扩大抽取范围。
